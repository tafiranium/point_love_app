## 1. 📦 Установка и запуск офлайн-версии

1. Скачайте или склонируйте папку `Client` (корень расширения).  
2. Откройте Chrome → `chrome://extensions/`.  
3. Включите **«Режим разработчика»** (переключатель в правом верхнем углу).  
4. Нажмите **«Загрузить распакованное расширение»** → выберите папку `Client`.  
5. Перезагрузите страницу `https://tl.myvirtualpos.ru/console/*` (или `/cpanel/console?path=/*`).  
6. Расширение начнёт работать – появятся логи в консоли страницы (F12).

## 2. 🗂 Структура проекта (важные файлы)

```plaintext
Client/
├── manifest.json                     # Точка входа: permissions, content_scripts, world: MAIN
├── app/
│   ├── defaultConfig.js              # DEFAULT_CONFIG – эталонные настройки
│   └── classes/
│       ├── configManager.js          # Синглтон для работы с конфигом (storage/память)
│       ├── monitorClass.js           # Центральный диспетчер – смотрит URL, запускает App или SaleManager
│       ├── appClass.js               # Класс App – для чековой страницы (view)
│       ├── manageClass.js            # Класс SaleManager – для списка продаж (manage)
│       └── assets/                   # Вспомогательные модули (interface, analys, tables, copy, type, comment...)
└── web/
    ├── background.js                 # Service worker (пока в основном для пересылки сообщений)
    ├── popup.html, popup.js, popup.css  # Интерфейс настроек
    └── icons/
```

### Главные классы

| Класс | Где живёт | Что делает |
|-------|-----------|-------------|
| `Monitor` | `monitorClass.js` | На странице: определяет `view` (чек) или `manage` (список продаж) |
| `App` | `appClass.js` | Работает на странице чека, проверяет поля, отправляет в Google-таблицу |
| `SaleManager` | `manageClass.js` | На странице списка продаж, открывает каждую продажу в фоне, проверяет ошибки |
| `ConfigManager` | `configManager.js` | Хранит и даёт конфиг всем остальным |

## 3. 🧩 Как добавить новый модуль (например, для печати)

1. Создайте новый файл в `app/classes/assets/printHelper.js`.  
2. Внутри напишите класс или функцию, которая будет выполнять нужное действие на странице.  
3. Подключите файл в `manifest.json` в массив `"js"` **перед** `monitorClass.js`.  
4. В `monitorClass.js` добавьте новый тип страницы (если требуется):  
   - Расширьте `this.url_template`, добавив новый шаблон, например `print: ["sales/print"]`.  
   - В `routeByUrl()` добавьте условие и вызов вашего модуля.  
5. Если модулю нужны настройки – добавьте поля в `DEFAULT_CONFIG` (в `defaultConfig.js`) и создайте UI в popup.

## 4. 🐞 Отладка

### 🔍 Логи content script (внедрённого на страницу)
- Откройте DevTools на самой странице myVirtual POS (F12) → вкладка **Console**.  
- Ищите логи с префиксами: `🚀`, `✅`, `❌`, `🔁`, `📋` – их выводит Monitor и другие классы.

### ⏸ Остановка и пошаговая отладка
- Поставьте `debugger;` в нужном месте кода.  
- Используйте точки останова непосредственно в исходном коде (вкладка Sources → `app/classes/...`).

### ⚙️ Отладка ConfigManager
В консоли страницы выполните:
```javascript
(await window.ConfigManager.getInstance()).getConfig()
```
Если `ConfigManager` не виден – убедитесь, что вызов `getInstance()` произошёл после загрузки скриптов.

### 🪟 Отладка popup
Кликните иконку расширения → правой кнопкой мыши по popup → **«Просмотреть элемент»** (откроется DevTools для popup).

### 🔧 Отладка background service worker
В `chrome://extensions/` найдите ваше расширение → нажмите **«service worker»** (синяя ссылка).

## 5. ⚠️ Частые проблемы и их решение

| Проблема | Вероятная причина | Решение |
|----------|------------------|---------|
| Расширение не запускается на странице | URL не совпадает с `matches` в `manifest.json` | Проверьте `matches` – там должно быть точно `https://tl.myvirtualpos.ru/console/*` и `/cpanel/...` |
| `configure is not defined` в логах | `ConfigManager` не записал конфиг в глобальную переменную | Добавьте в `configManager.js` после загрузки: `window.configure = this.#cachedConfig`. Или `window.configure = await ConfigManager.getInstance().getConfig()`. |
| `SaleManager is not defined` | `manageClass.js` не подключён в манифесте | Добавьте `"app/classes/manageClass.js"` в `content_scripts` → `"js"` (перед `monitorClass.js`). |
| `App` не находит `interface.remove()` | У `App` нет свойства `app.interface` или структура другая | Проверьте `appClass.js`. Возможно, `App` возвращает экземпляр с методом `destroy()` или `remove()`. Скорректируйте `startMainApp()` в `monitorClass.js`. |
| Таблица на странице `manage` не обрабатывается | `SaleManager` не создаётся из-за пустой таблицы | Логи в `attemptRestart()` покажут `Таблица пуста, повтор...`. Увеличьте задержки или количество попыток. |
| Конфиг не сохраняется в `chrome.storage` | Нет permission `"storage"` в манифесте | Добавьте `"permissions": ["storage"]` (уже есть, но проверьте). Также убедитесь, что вы не запускаете скрипт вне расширения. |

## 6. ⚙️ Где записаны настройки (конфиг)

- **Эталон** – `app/defaultConfig.js` (объект `DEFAULT_CONFIG`).  
- **Сохранённый пользователем** – `chrome.storage.local['config']` (обновляется через popup).  
- **Текущий в работе** – кеш внутри `ConfigManager`. При старте загружается из storage или DEFAULT_CONFIG.

Если хотите принудительно сбросить настройки к эталону:
```javascript
(await ConfigManager.getInstance()).resetConfig();
```

## 7. 🔄 Планируемое добавление клиентской версии (GitHub-синхронизация)

В будущем код будет минифицироваться и собираться в один бандл, а `DEFAULT_CONFIG` подгружаться с GitHub. Для этого уже есть `github.js` / `github.py` (пока не активно).  

**Принцип:**  
- Расширение проверяет версию конфига на GitHub.  
- Если новее – скачивает и сохраняет в `chrome.storage.local`.  
- Офлайн-разработка не затронет этот механизм.
